================================================================================
                           THE SOLUTION (ATTEMPT #72)
================================================================================

THE MAGIC FORMULA:
------------------
keyboardPadding = WindowInsets.ime.bottom - 130.dp

Where 130.dp accounts for:
- Bottom navigation bar (~80.dp)
- System navigation bar (~50.dp on this device)

IMPLEMENTATION:
---------------
1. In ChatDetailScreen.kt (lines ~86-92):

    // KEYBOARD FIX: Calculate height above bottom nav
    val imeInsets = WindowInsets.ime
    val density = LocalDensity.current
    val imeBottom = with(density) { imeInsets.getBottom(density).toDp() }
    val bottomNavHeight = 130.dp  // <-- THE MAGIC NUMBER!
    val keyboardHeight = (imeBottom - bottomNavHeight).coerceAtLeast(0.dp)

2. Apply padding to input Surface (line ~315):

    Surface(
        modifier = Modifier
            .fillMaxWidth()
            .padding(bottom = keyboardHeight),  // <-- KEYBOARD AWARE PADDING
        color = Color.Black,
        tonalElevation = 0.dp
    ) {
        // Input field, attachment button, send button
    }

================================================================================
                         KEY INSIGHTS
================================================================================

1. The Android keyboard is notoriously difficult to handle in Compose
2. adjustResize + manual insets calculation is the most reliable approach
3. The magic number (130.dp) varies by device but can be tuned
4. Bottom nav must stay visible - hiding it breaks the calculation
5. Subtracting the combined height of all bottom UI is the key
